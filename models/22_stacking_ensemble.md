# Stacking / Soft-Voting Ensemble

> **One-liner:** A meta-learner (or simple average) that combines the folder's diverse base models into one prediction — usually the best raw scores, at the cost of interpretability.
> **Category:** method ·
> **Runs on:** local CPU ·
> **Priority:** stretch ·
> **Interpretable:** no (reduced — the individual base explanations are diluted)

## Why it fits Genome Firewall
The base models in this folder capture different structure in the sparse binary AMR matrix — L2 logistic regression captures additive marker effects, gradient boosting captures gene×gene interactions, Bernoulli naive Bayes captures per-marker likelihoods, and the ESM-2 head captures sequence context the presence/absence table misses. Because these errors are partly uncorrelated, combining them can squeeze out extra balanced accuracy and, more importantly for this project, a better-calibrated combined score after a final calibration step. On a small, imbalanced, per-antibiotic dataset an ensemble also tends to be steadier across genetic groups than any single learner. This is a stretch goal precisely because it trades away the honest, one-coefficient-per-gene explanation that the judging values.

## When to prefer it / when to skip it
Prefer stacking only after the base models are individually solid and you have confirmed on the held-out grouped-test Brier that no single model already dominates — the ensemble must *earn* its complexity. Use **soft voting** (a plain probability average, optionally weighted) as the lighter alternative when you want most of the robustness benefit with far less leakage surface and almost no tuning. **Skip the stack, or keep it strictly secondary, if it does not beat the calibrated L2 logistic baseline on test Brier by a margin that survives the per-genetic-group breakdown** — a fancier model that cannot beat the interpretable baseline is evidence the complexity is unjustified. Critically: even when the stack scores highest, **keep the logistic-regression baseline as the reported explainable model** (rigor rule 5); ship the ensemble as the score, the logistic model as the explanation, and be explicit about which is which.

## Data interface (the contract this code must respect)
- Base learners each read `data/processed/features.parquet` (and the ESM-2 head its sequence inputs), fit on the **train** split ONLY.
- **Meta-features are generated by out-of-fold prediction using GroupKFold on `cluster_id` WITHIN the train split only** — never use `cal` or `test` to build meta-features.
- The meta-learner is trained on those out-of-fold meta-features (train only).
- The **final stack is calibrated on the `cal` split ONLY**; all metrics are reported on the **`test`** split ONLY, and per `cluster_id` genetic group.
- Reads `db/drugs_saureus.csv` for the deterministic target gate.

## Adversarial checks it must survive
- **Meta-feature leakage (Rule 1) — the critical one:** if the meta-learner is trained on base-model predictions made *on the same rows the base models trained on*, those predictions are optimistic and the stack overfits catastrophically. Generate meta-features via **out-of-fold GroupKFold on `cluster_id` inside train** so every meta-feature is a genuine held-out prediction, and no cluster ever helps predict itself. Confirm no `cluster_id` spans a fold's train/holdout boundary.
- **Never touch cal/test for meta-features:** the `cal` split is reserved for calibrating the final stack, and `test` is untouched until reporting — using either to build meta-features leaks.
- **Clonal memorization / generalization (Rule 6):** an ensemble has high capacity and can memorize *S. aureus* lineage structure; judge it on the per-genetic-group breakdown including unseen clusters, and expect a larger drop there than the simpler baselines show.
- **Calibration (Rule 2):** stacking can produce sharper but miscalibrated scores; calibrate the final combined probability on `cal` and report Brier + reliability on `test`.
- **Interpretability loss (Rule 5):** meta-learner weights over base models are not gene-level biological explanations. Do not present them as evidence; retain the logistic baseline as the explainable model and label the stack's output as a statistical combination only.
- **Target gate (Rule 4):** the ensemble's "S" must still pass the target gate — absence of markers alone cannot yield "likely to work."

## Hyperparameters worth sweeping
- **Base-learner roster:** which models to include (e.g. L2 logistic + HistGradientBoosting + Bernoulli NB + ESM-2 head); diversity matters more than adding more of the same family.
- **Meta-learner:** L2 logistic regression (recommended — few parameters, less overfit on small data) vs its `C`; keep it deliberately simple.
- **Meta-feature form:** predicted probabilities vs `passthrough` of a few raw features; probabilities-only is safest on small data.
- **Number of GroupKFold folds** on `cluster_id` within train (e.g. 5), constrained by how many clusters exist in train.
- **Voting weights** (soft-voting alternative): uniform vs Brier-weighted per base model, tuned on the train out-of-fold scores only.
- **Final calibration method** on `cal`: sigmoid/Platt vs isotonic (prefer Platt when `cal` is small — see the calibration-methods file).

## Calibration & no-call handling
Assemble the stack (`StackingClassifier` with `cv=GroupKFold(...)` on `cluster_id`, or a hand-rolled out-of-fold routine, plus a logistic meta-learner) or the soft-voting average, all fit within train. Then wrap the final combined model with `CalibratedClassifierCV(cv="prefit")` fit on the **cal** split to make the combined probability honest. That calibrated probability of `R` feeds the shared no-call logic: return `no-call` when calibrated p is in the ambiguous band (~0.4–0.6), when the genome is out-of-distribution (unseen cluster / markers unlike training), or when the target gate fires. Report the reliability diagram before and after final calibration.

## Metrics to report
On the held-out grouped **test** split, and per genetic group (including unseen clusters): balanced accuracy, recall_R, recall_S, F1, AUROC, PR-AUC, **Brier score**, **reliability diagram**, and no-call rate + accuracy-on-called. Report every metric as a **delta versus the L2 logistic baseline** so the ensemble's added complexity is judged explicitly, and expect/report a larger unseen-cluster drop. Save per-antibiotic metrics to JSON and print a summary table.
